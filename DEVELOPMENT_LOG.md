# 開發日誌 (Development Log) - 台南綠園道共創 (Tainan Greenway Co-Creation)

## 里程碑 03：手繪水彩風格公民 RPG 與圓桌互動對話流 (Milestone 03: Watercolor RPG & Interactive Seating Chat)
**日期**：2026-05-29

### 1. 當前已完成功能 (Completed Features)
- **手繪水彩風格插圖整合**：
  - 導入了我們利用繪圖工具生成的 `/greenway_watercolor.png`，作為首頁主視覺和規劃地圖頂部的精緻景觀橫幅。
  - 在結果頁頂部引入了 2.5D 等角水彩插畫風未來城市願景圖 `/future_city_25d.png`，展示協商完成後的綠帶想像。
  - 在任務簡報頁面的三個衝突點，分別嵌入對應手繪圖 `/conflict_privacy.png`、`/conflict_commerce.png`、`/conflict_ecology.png`，視覺效果極佳。
- **角色插畫形象與自我介紹對話氣泡**：
  - 身分選擇、指標面板與滿意度矩陣，全面替換成 6 大市民代表的水彩手繪頭像圖片（`/avatar_*.png`）。
  - 將關係人卡片的身分細節重構為指向其頭像的 HTML 對話框氣泡，模擬角色正在對玩家開口自我介紹的 RPG 臨場感。
- **寫生繪本選單首頁**：
  - 首頁改版為 RPG 遊戲開始選單樣式，附帶 `開始新局`、`載入存檔（鎖定）` 與可開啟手繪風對話引導彈窗的 `遊戲說明`。
- **圓桌會議聊天流與真實互動**：
  - 將協商圓桌背景重構為綠色水彩渲染的有機草坪 blob，並為席次加上動態高亮發言狀態。
  - 改建討論區為時間線對話紀錄：進入回合時先展示各代表的初始主張；玩家選定選項後，以玩家角色（如阿明）名義動態將「提案對話泡」加入聊天流；隨後觸發模擬 AI 評議，以對話氣泡形式呈現其他 NPC 對此決策的動態反饋，大幅增強了故事性與帶入感。
- **筆刷質感指標條**：
  - 都市指標進度條套用波浪筆觸（Brush Stroke）效果，並搭配草寫風數值變化標籤。

### 2. 已知問題與限制 (Known Issues & Limitations)
- **AI 回應為本地動態生成**：目前 NPC 對決策的回應與態度在 `mockResponses.ts` 中依選項 ID 對照動態渲染，尚未真正串接即時 LLM。
- **離線狀態**：所有玩家決策影響與歷史記錄仍存於 React 狀態中，無後端持久化儲存。

### 3. 下一步規劃 (Next Steps)
- **伺服器 API 路由**：實作 `/api/negotiate` 路由，將當前 roundId、playerRoleId、choiceId 發送至後端，並呼叫真實 Gemini/OpenAI API 流式返回生成的利益關係人發言。

## 里程碑 02：溫暖遊戲化公民協商儀表板 (Milestone 02: Soft Civic Game Dashboard Redesign)
**日期**：2026-05-29

### 1. 當前已完成功能 (Completed Features)
- **視覺風格全面改版**：
  - 將原本硬朗的黑白工程圖紙、CAD座標線與圖紙代碼移除，全面改為溫暖的米黃/暖白背景（`var(--color-bg-warm)`），並搭配柔和的暖褐色網點背景（`radial-gradient`）。
  - 使用大圓角（`rounded-2xl`、`rounded-3xl`）與暖色調投影（`shadow-soft`、`shadow-soft-sm`），讓卡片與區塊富有空氣感與遊戲介面氛圍。
- **角色扮演卡片化**：
  - 協商身分選擇卡片新增對應圖示（Home、Store、Bike、Heart、Trees、Building2）與多彩圓形背景，核心價值以獨立膠囊標籤呈現。
- **協商圓桌對話框**：
  - 將原本的對話列表重構為具備箭頭尾部的漫畫風對話氣泡（`chat-bubble-left`），呈現圍繞圓桌的模擬 AI 市民意見交流。
- **空間地圖優化**：
  - 左側都市計畫剖面圖軸線平滑化，並配以對應代表色的彩色段落，以精緻的視覺效果呈現分段式空間策略。
- **績效與滿意度矩陣**：
  - 五大績效指標條採用柔和的主題色（珊瑚橘、水藍、生態綠、琥珀黃），並以括號標示變動數值（如 `+15` 或 `-10`）。
  - 結果頁新增 6 大市民代表頭像、代表身分與滿意度表情符號（😊/😐/😞）組成的矩陣網格，視覺反饋更加直觀。

### 2. 已知問題與限制 (Known Issues & Limitations)
- **AI 對話為本地模擬**：目前各利益關係人的發言與態度仍為本地靜態 mock 數據。
- **非同步延遲載入**：對話切換時使用 900ms 的非同步延遲與載入狀態以模擬真實 API 對答過程。

### 3. 下一步規劃 (Next Steps)
- **串接真實 LLM API**：在 `src/lib/api.ts` 中實作真實的 POST 請求至 `/api/negotiate`，並在後端路由中動態生成利益關係人發言。

## 里程碑 01：當前運作原型 (Milestone 01: Current Working Prototype)
**日期**：2026-05-29

### 1. 當前已完成功能 (Completed Features)
- **公民角色扮演系統**：支持選擇 6 組利益關係人身分憑證（居民、在地店家、通勤與自行車族、高齡步行者、環保團體、市府與景觀設計師），每組角色擁有獨特的價值觀、關注議題與代表發言。
- **公民協商圓桌機制**：模擬三回合空間協商對話。每回合動態加載其他利益關係人對特定空間議題的觀點與意見。
- **解耦模擬 AI 引擎**：透過模組化非同步調用函式（模擬網路延遲與加載狀態），展現 AI 對話引擎模擬中的界面效果，為未來串接真實 LLM 伺服器端路由做好了準備。
- **二維都市計畫圖紙 (動態地圖)**：左側抽象剖面圖可隨玩家在協商過程中的決策（如：高架自行車道、降至地面、增加隱私遮簾等）即時更新空間型態與對應符號（斜線網格、綠帶符號、立體交叉標記）。
- **衝擊性指標評估系統**：右側儀表板動態反映 5 大指標（居住舒適、商業活力、交通效率、生態棲地、歷史記憶）在決策後的數值起伏，當指標低於 35 時觸發紅色警報，高於 70 時觸發綠色最佳配置標籤。
- **共創審定備忘錄**：在協商完成後，彙整玩家的變更策略、績效指標評估，並生成包含公民小組與景觀委員會的代表聯署公文。

### 2. 當前 UI 狀態 (UI Status)
- **建築評圖板美學 (Presentation Board)**：全螢幕直角無陰影設計，配有 20px 與 100px 雙重灰線網格背景與定位十字 `+`。
- **等寬字型標註 (Monospace Annotations)**：所有標題、比例尺、座標、里程（例如 `STA 0+700`）與狀態均採用 monospace 字型呈現，模擬 CAD 圖紙。
- **中文介面化**：所有內容文字、標籤、選項與提示 100% 實現中文本土化，視覺一致性高。

### 3. 已知問題與限制 (Known Issues & Limitations)
- **AI 對話為靜態模擬**：當前利益關係人發言為本地預設數據，尚無真實生成式對話。
- **離線狀態**：所有玩家決策影響與策略判定皆在前端瀏覽器內計算，無後端數據持久化。
- **僅支援桌面版**：本原型專為桌面大螢幕評圖展示設計，未對行動裝置進行響應式適配。

### 4. 下一步規劃 (Next Steps)
- **Next.js 伺服器路由串接**：在 `src/lib/api.ts` 中以 `fetch('/api/negotiate')` 替代模擬的非同步調用，在後端使用 real-time LLM API 生成動態的利益關係人回應。
- **空間策略擴展**：加入更多協商回合、都市規劃議題與分區段空間選項。
- **協商歷史記錄保存**：支持導出 PDF 格式的公民協商備忘錄或規劃圖紙。
